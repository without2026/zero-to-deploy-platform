# zero-to-deploy-platform

Reusable CI/CD workflow library + runtime assets for the zero-to-deploy pipeline.

## What lives here

- `.github/workflows/*.yml` — reusable workflows (`on: workflow_call:`) consumed by generated apps
- `.github/scripts/*.py` — helper scripts invoked by workflows (cross_validator, coverage-delta, etc.)
- `pipeline/core/` — runtime assets shared by all adapters: `authority.yml`, `spec-schema.json`, `spec-coverage.py`, `assert-guard.py`, `judgment-prompt.md`
- `pipeline/adapters/{android,electron,web}/` — per-platform build/test/gates/deploy config

## How apps consume this

Apps generated by the `zero-to-deploy` skill pin to a version tag (`@v1`) and call the workflows via `workflow_call`:

```yaml
jobs:
  sast:
    uses: without2026/zero-to-deploy-platform/.github/workflows/security-sast.yml@v1
    with:
      adapter: android
    secrets: inherit
```

## Versioning

SemVer. See `VERSION.md` for the changelog and breaking-change policy.

- `@v1` — floating major (recommended default)
- `@<sha>` — immutable pin (recommended for high-security consumers)
